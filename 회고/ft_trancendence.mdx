## Keep

- 3주라는 짧은 시간에 개발의 집중도를 올릴 수 있었다.

  - 짧은 개발기간으로 인한 기능 우선 개발로 방향을 잡고 빠르게 개발 할 수 있었다.

- 웹 소켓을 이용한 게임 구현

  - 웹 소켓을 이용해 PONG 게임을 구현했다.
  - 웹 소켓을 어떻게 쓰는지 알게 된 좋은 프로젝트였고, 또한 소켓을 이용함에 있어서 개발자가 주의를 가져야 하는 부분들도 직접적으로 알 수 있었다.

- 백엔드의 코드 수정
  - 이 프로젝트에서는 프론트엔드를 혼자서 작업을 진행했지만, 짧은 개발 기간이라는 제한때문에 백엔드 분들의 코드도 내가 직접 고쳐가며 서비스를 만든 경우가 많았다.

## Problem

- 짧은 기간으로 인한 아쉬움

  - 짧은 기간을 잡게 된 이유가 팀원분 중 한분이 독일로 유학을 가야하는 이유가 있었다. 그렇기 때문에 약간 챌린지의 느낌처럼 프로젝트를 진행하게 되었다.
  - 짧은 기간덕에 정말 우선적으로 기능이 되게끔만 먼저 만들자라는 셍각으로 만들다 보니 코드의 퀄리티가 다소 많이 떨어지는 게 많이 아쉽긴 하다....

- 풀스택으로 작업을 진행
  - 백엔드 2명, 프론트엔드 1명으로 백엔드 분들은 웹서비스를 처음 만들어보는 분들이시다 보니 러닝커브가 확실히 존재했고, 나는 그래도 그 분들보다는 웹 프로젝트를 해본 경험이 있었기 때문에 API 문서라던지 화면 설계, 프로젝트 설계를 직접 하고, 프로젝트를 진행하면서 백엔드분들이 어려워하는 부분들은 또 내가 직접 코드를 수정해가며 잔행한게 아쉽다... 원래는 같이 공부를 하면서 모르는걸 알려주고, 내가 모르는걸 도움받는 식의 협업을 해야하는데 그런 협업이 거의 없이 진행이 된게 많이 아쉽긴하다...
